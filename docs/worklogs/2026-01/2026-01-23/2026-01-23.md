### C. 오늘의 문제 1개
- 문제/증상: GIS 용어와 툴(QGIS, GDAL, EPSG, DEM/DTM/DSM)이 서로 섞여 보여서 **실무에서 어떤 흐름으로 쓰는지**가 감이 안 잡힘
- 원인 가설:
  - 데이터 형식(래스터/벡터)과 처리 도구(QGIS/GDAL)가 같은 레벨의 개념처럼 느껴져 혼동됨
  - 좌표계(EPSG)와 거리/면적 측정 가능 여부가 명확히 연결되지 않아 실수 위험이 있음
  - DEM이 문서/기관에 따라 DTM/DSM을 통칭하는 등 용어 사용이 일관되지 않아 오해가 생김
- 확인 방법:
  - QGIS에서 shp(벡터) 레이어를 직접 불러오고 레이어 순서를 바꿔 시각적으로 확인
  - 좌표계(EPSG 코드) 예시를 기준으로 “경위도(4326) vs 평면좌표(T M 5186)” 차이를 정리
  - DEM/DTM/DSM은 정의와 용도를 표로 정리해 서로 비교
- 해결:
  - 개념을 “데이터 형식(래스터/벡터) → 도구(QGIS/GDAL) → 좌표계(EPSG) → 고도 모델(DEM/DTM/DSM)” 순으로 재배치해 학습 노트를 구성
  - QGIS에서 shp 적용 절차를 스크린샷 기반으로 단계화(레이어 추가 → 벡터 레이어 → shp 선택 / XYZ Tiles → OSM 추가 / 레이어 순서 조정)
- 결과:
  - GIS 데이터/툴/좌표계의 관계를 한 문장으로 정리 가능해짐:  
    - “사람은 QGIS로 보고 편집하고, 변환/재투영 같은 처리는 GDAL이 담당한다.”
  - shp 레이어를 불러오고 배경지도(OSM)와 함께 표시하는 기본 워크플로우를 확보
- 다음 액션:
  - QGIS에서 현재 레이어의 좌표계 확인/변경(프로젝트 CRS) 흐름을 추가로 정리
  - GDAL 명령(예: 형식 변환/재투영) 2~3개를 실제 예제로 실행해보고 worklog에 기록
  - 브이월드 좌표계 이슈(거리 측정/분석 적합 좌표계 선택) 관련해서 “언제 어떤 EPSG를 쓰는지” 기준을 한 페이지로 정리

### E. 키워드 3개
- 키워드: Raster vs Vector / QGIS & GDAL / EPSG(4326, 5186) + DEM/DTM/DSM
- 한 줄 설명:
  - Raster vs Vector: 래스터는 픽셀 그리드(확대 시 깨짐), 벡터는 점·선·면의 도형 정보(상대적으로 용량 효율적)
  - QGIS & GDAL: QGIS는 시각화·편집 GUI, GDAL은 공간데이터 읽기/쓰기/변환·재투영을 담당하는 라이브러리/CLI 도구
  - EPSG + DEM 계열: EPSG는 좌표계 코드(4326=경위도, 5186=한국 TM), DEM/DTM/DSM은 고도 모델로 용도와 포함 대상(지면/표면)이 다름

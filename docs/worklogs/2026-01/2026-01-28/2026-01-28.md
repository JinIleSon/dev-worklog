### C. 오늘의 문제 1개
- 문제/증상:
  - 사용자가 “어떤 파일을 어떤 기준으로 올려야 하는지”와 “검사가 무엇을 의미하는지(왜 불합격인지)”를 UI만 보고 이해하기 어렵고, 업로드 실패 시 재시도 흐름이 끊길 수 있음
- 원인 가설:
  - 업로드 제약(SHP 구성요소, 용량 제한, 단일 지하시설물 제출, 맨홀/밸브 업로드 규칙)이 화면에서 즉시 이해되지 않으면 실패/재시도 반복이 발생함
  - 파일명 규칙과 레이어 정의(레이어ID/레이어명)가 분리돼 있으면 사용자가 업로드 전 확인을 놓칠 수 있음
  - 검사 결과가 “합격/불합격”만 보이면, 어떤 규칙(속성/공간/위상/연결성)에서 문제가 났는지 액션으로 연결이 어려움
- 확인 방법:
  - 업로드→실패 모달→가이드 확인→재업로드까지 사용자가 “다음 행동”을 찾을 수 있는지 동선 점검
  - 속성검사/공간검사 결과에서 ‘상세 내용(분류/내용)’이 사용자 수정 포인트로 충분한지 점검
  - 공간검사 항목 선택(예: 자기교차, 중복객체, 관로접합 등)과 진행 상태(예상 소요 시간/합격/불합격)가 한 화면에서 이해되는지 점검
- 해결:
  - 업로드 화면 우측에 “안내 사항”을 고정 배치하고, 같은 영역에 “파일명 규칙(분류 탭+레이어 테이블)”을 함께 제공하는 구성으로 확정
  - 업로드 실패 시 모달에서 “우측 안내사항 및 파일명 가이드라인 참고 후 재시도” 메시지로 리커버리 동선을 명시
  - 품질 검수 화면을 2단(속성 정보 품질 검사 / 지하 시설물 공간 정보 품질 검사)으로 분리하고, 결과는 카드+상세 모달+검사 항목 선택/진행 상태까지 제공하도록 확정
- 결과:
  - 사용자의 행동 흐름이 명확해짐: **업로드(제약 확인) → 파일명 규칙 확인 → 분류/지역 선택 → 속성검사 결과 확인(상세) → 공간검사 항목 선택/진행 → 최종 결과 확인**
  - “왜 불합격인지”를 상세 모달/항목 단위로 내려줄 수 있는 UI 기반이 마련됨(운영/가이드 문구 확장 여지 포함)
- 다음 액션:
  - 개발 전달용 산출물 정리: 화면별 컴포넌트 목록/상태(초기/로딩/실패/성공) 정의
  - 오류/결과 데이터 스키마 초안 작성(오류코드, 심각도, 분류, 객체ID/좌표, 해결가이드, 오탐/미탐 처리)
  - 검사 항목 선택 UI에 대한 정책 확정(기본 선택 세트, 오차범위 입력 단위/기본값, 항목 설명 툴팁)
